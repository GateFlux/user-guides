User Management Guide

Learn how to manage users, roles, permissions, and resident profiles in GateFlux.

Overview

This guide covers user management in GateFlux, including creating users, managing roles and permissions, and handling resident profiles.

User Types

GateFlux manages several types of users:

System Users

Super Admins, Society Admins, Committee Members with admin panel access.

Residents

Flat Owners, Tenants, and Family Members residing in the society.

Operational Users

Security Guards and Society Staff managing daily operations.

Managing System Users

Creating a New Admin User

Via API:

HTTP 
POST /api/admin/users
Authorization: Bearer <admin_token>
Content-Type: application/json

{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "SecurePassword123",
    "role": "society_admin",
    "society_id": 1
}

Via Web Dashboard:

  1. Navigate to Settings → User Management
  2. Click Add New User
  3. Fill in user details
  4. Assign appropriate role
  5. Click Save

User Roles and Permissions

GateFlux implements a comprehensive Role-Based Access Control (RBAC) system with 12 predefined role templates and 120+ granular capabilities.

Assigning Roles

HTTP 
POST /api/v1/users/{user_id}/roles
Authorization: Bearer <token>
Content-Type: application/json

{
    "role_template": "treasurer",
    "society_id": "uuid"
}

Check User Capabilities

HTTP 
GET /api/v1/me/capabilities
Authorization: Bearer <token>

Response:

JSON 
{
    "success": true,
    "data": {
        "role": "treasurer",
        "level": 70,
        "capabilities": [
            "finance.view",
            "finance.approve",
            "finance.budget_manage",
            "finance.collections_view",
            "expenses.view",
            "expenses.approve"

    }

Managing Residents

Adding a New Resident

Residents are managed through the Flat/Unit system. Each flat can have:

  • One Primary Owner/Tenant
  • Multiple Family Members
HTTP 
POST /api/societies/{society_id}/flats/{flat_id}/residents
Authorization: Bearer <token>
Content-Type: application/json

{
    "name": "Resident Name",
    "email": "resident@example.com",
    "phone": "+91-9876543210",
    "type": "owner",
    "is_primary": true,
    "move_in_date": "2024-01-15"
}

Resident Profile Fields

Field Type Required Description
name string Yes Full name of resident
email string Yes Email address (must be unique)
phone string Yes Contact number
type enum Yes owner, tenant, family_member
is_primary boolean No Primary contact for the flat
move_in_date date No Date of moving in
id_proof_type string No Aadhaar, PAN, Passport, etc.
id_proof_number string No ID document number

{

    "phone": "+91-9876543211",

Transferring Flat Ownership

When a flat changes hands:

  1. Navigate to the flat details
  2. Click Transfer Ownership
  3. Mark current residents as moved out
  4. Add new owner/tenant details
  5. Update vehicle and domestic help registrations
HTTP 
POST /api/flats/{flat_id}/transfer
Authorization: Bearer <token>
Content-Type: application/json

{
    "move_out_date": "2024-06-30",
    "new_resident": {
        "name": "New Owner",
        "email": "newowner@example.com",
        "phone": "+91-9876543212",
        "type": "owner"
    }
}

Two-Factor Authentication (2FA)

GateFlux supports TOTP-based Two-Factor Authentication for enhanced account security.

Enabling 2FA

  1. Navigate to Profile → Security or /security/two-factor
  2. Click "Enable Two-Factor Authentication"
  3. Scan the QR code with your authenticator app (Google Authenticator, Authy, Microsoft Authenticator)
  4. Enter the 6-digit verification code from your app
  5. Save your recovery codes in a secure location

2FA API Endpoints

HTTP 
# Check 2FA status
GET /api/v1/auth/2fa/status
Authorization: Bearer <token>

# Start 2FA setup (generates QR code)
POST /api/v1/auth/2fa/setup
Authorization: Bearer <token>

# Confirm and enable 2FA
POST /api/v1/auth/2fa/confirm
Authorization: Bearer <token>
Content-Type: application/json
{
    "code": "123456"
}

# Disable 2FA (requires password and code)
POST /api/v1/auth/2fa/disable
Authorization: Bearer <token>
Content-Type: application/json
{
    "password": "current_password",
    "code": "123456"
}

Logging In With 2FA

When 2FA is enabled, login requires an additional verification step:

HTTP 
# Step 1: Normal login returns requires_2fa flag
POST /api/v1/auth/login
{
    "email": "user@example.com",
    "password": "password"
}

Response:
{
    "data": {
        "requires_2fa": true,
        "user_id": "uuid-here"
    }
}

# Step 2: Verify with 2FA code
POST /api/v1/auth/2fa/verify
{
    "user_id": "uuid-here",
    "code": "123456"
}

# Or use recovery code if authenticator unavailable
POST /api/v1/auth/2fa/verify
{
    "user_id": "uuid-here", 
    "recovery_code": "ABCD-1234"
}

Recovery Codes

Recovery codes are generated when 2FA is enabled. Each code can only be used once. To regenerate:

HTTP 
POST /api/v1/auth/2fa/recovery-codes/regenerate
Authorization: Bearer <token>
Content-Type: application/json
{
    "code": "123456"
}

Best Practices

Security Best Practices
  • Enable Two-Factor Authentication - Highly recommended for all admin users
  • Use strong passwords - Minimum 8 characters with mixed case, numbers, and symbols
  • Regular audits - Review user access quarterly
  • Immediate deactivation - Disable accounts for moved-out residents
  • Verify contact information - Ensure phone numbers and emails are current
  • Photo verification - Maintain updated photos for residents and staff
