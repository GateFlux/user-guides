Governance & Elections Guide

Manage AGMs, digital voting, elections, and nominations for your housing society.

Owner-Only Access: Governance features are restricted to unit owners. Tenants cannot access voting, elections, or audit documents.

Overview

GateFlux provides comprehensive governance tools for housing societies, including:

  • AGM Management - Schedule, publish notices, track RSVP and quorum
  • Digital Voting - Secure voting on motions and resolutions
  • Elections - Committee elections with nomination workflow
  • Audit Access - Financial document access for owners

Owner vs Tenant Access

Access to governance features is determined by ownership type:

Feature Owner Co-Owner Tenant
View AGM Details ✅ ✅ ❌
Cast Votes ✅ ✅ ❌
Contest Elections ✅ ✅ ❌
View Audit Ledger ✅ ✅ ❌
File Nominations ✅ ✅ ❌

AGM Management

Creating an AGM (Secretary)

Only users with governance.agm_manage capability can create AGMs.

HTTP 
POST /api/v1/governance/admin/agms
Authorization: Bearer <token>
Content-Type: application/json

{
    "title": "Annual General Meeting 2026",
    "meeting_date": "2026-03-15",
    "meeting_time": "10:00",
    "venue": "Society Clubhouse, Ground Floor",
    "agenda": [
        "Approval of previous meeting minutes",
        "Financial report presentation",
        "Election of new committee members",
        "Maintenance fee revision proposal"
    ],
    "quorum_percentage": 51,
    "rsvp_deadline": "2026-03-10"
}

Publishing AGM Notice

Once created, publish the AGM notice to notify all owners:

HTTP 
POST /api/v1/governance/admin/agms/{agm_id}/publish-notice
Authorization: Bearer <token>

Owner RSVP

Owners can RSVP for the AGM or assign a proxy:

HTTP 
POST /api/v1/governance/agms/{agm_id}/rsvp
Authorization: Bearer <token>
Content-Type: application/json

{
    "attending": true,
    "proxy_to": null
}

Digital Voting

Voting Principles

  • One Vote Per Unit - Each unit gets one vote, regardless of co-owners
  • Vote Secrecy - Individual votes are hashed, only aggregates are shown
  • Real-time Quorum - Quorum percentage tracked in real-time
  • Immutable Record - Votes cannot be changed once cast

Creating a Motion (Committee)

HTTP 
POST /api/v1/governance/admin/motions
Authorization: Bearer <token>
Content-Type: application/json

{
    "agm_id": "uuid-of-agm",
    "title": "Proposal to Install Solar Panels",
    "description": "Install 50kW solar panel system on building rooftop to reduce electricity costs by an estimated 30%.",
    "motion_type": "special",
    "seconder_id": "uuid-of-seconder",
    "requires_quorum_percentage": 67
}

Motion Types

Type Quorum Required Description
ordinary 51% Routine decisions, maintenance approvals
special 67% Major expenses, bylaw changes
procedural 51% Meeting procedures, adjournment

Opening Voting on a Motion

HTTP 
POST /api/v1/governance/admin/motions/{motion_id}/open-voting
Authorization: Bearer <token>
Content-Type: application/json

{
    "voting_ends_at": "2026-03-15T18:00:00Z"
}

Casting a Vote (Owner)

HTTP 
POST /api/v1/governance/motions/{motion_id}/vote
Authorization: Bearer <token>
Content-Type: application/json

{
    "vote": "for",
    "unit_id": "uuid-of-unit"
}

Vote Options:

  • for - Vote in favor of the motion
  • against - Vote against the motion
  • abstain - Abstain from voting

Viewing Results

HTTP 
GET /api/v1/governance/motions/{motion_id}/results
Authorization: Bearer <token>

Response:

JSON 
{
    "success": true,
    "data": {
        "motion": {
            "id": "uuid",
            "title": "Proposal to Install Solar Panels",
            "status": "passed"
        },
        "results": {
            "votes_for": 45,
            "votes_against": 12,
            "votes_abstain": 3,
            "total_votes": 60,
            "quorum_met": true,
            "passing_percentage": 75.0
        }
    }
}

Elections

Creating an Election (Secretary)

HTTP 
POST /api/v1/governance/admin/elections
Authorization: Bearer <token>
Content-Type: application/json

{
    "agm_id": "uuid-of-agm",
    "position": "President",
    "description": "Election for Society President for term 2026-2028",
    "max_winners": 1,
    "nomination_starts_at": "2026-02-15T00:00:00Z",
    "nomination_ends_at": "2026-03-01T23:59:59Z",
    "voting_starts_at": "2026-03-15T10:00:00Z",
    "voting_ends_at": "2026-03-15T18:00:00Z"
}

Filing a Nomination (Owner)

HTTP 
POST /api/v1/governance/elections/{election_id}/nominate
Authorization: Bearer <token>
Content-Type: application/json

{
    "statement": "I have been a resident for 10 years and served on the security committee. I am committed to improving our society's infrastructure and financial management.",
    "seconder_id": "uuid-of-seconder"
}

Nomination Workflow

  1. Owner files nomination during nomination period
  2. Nomination status is pending
  3. Secretary reviews and approves/rejects
  4. Approved nominations appear on ballot
  5. Voting opens on scheduled date

Approving Nominations (Secretary)

HTTP 
POST /api/v1/governance/admin/nominations/{nomination_id}/approve
Authorization: Bearer <token>

Casting Election Vote

HTTP 
POST /api/v1/governance/elections/{election_id}/vote
Authorization: Bearer <token>
Content-Type: application/json

{
    "nomination_id": "uuid-of-candidate-nomination",
    "unit_id": "uuid-of-unit"
}

Election Results

HTTP 
GET /api/v1/governance/elections/{election_id}/results
Authorization: Bearer <token>

Check Voting Status

Owners can check their voting eligibility and pending votes:

HTTP 
GET /api/v1/governance/my-voting-status
Authorization: Bearer <token>

Response:

JSON 
{
    "success": true,
    "data": {
        "is_owner": true,
        "is_tenant": false,
        "is_committee_member": true,
        "can_vote": true,
        "can_contest_election": true,
        "owned_units": [
            {
                "unit_id": "uuid",
                "unit_number": "A-101",
                "ownership_type": "owner"
            }
        ],
        "pending_votes": [
            {
                "motion_id": "uuid",
                "motion_title": "Solar Panel Installation",
                "unit_id": "uuid",
                "voting_ends_at": "2026-03-15T18:00:00Z"
            }
        ]
    }
}

Audit Ledger Access

Owners have read-only access to financial audit documents:

HTTP 
GET /api/v1/governance/audit-ledger
Authorization: Bearer <token>
Committee vs Owner Access: Committee members have full export rights, while regular owners have read-only access to audit documents.
