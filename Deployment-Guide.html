1. Domain Configuration

1.1 API (Laravel)

api/.env

Update these environment variables:

# Application URL
APP_URL=https://api.yourdomain.com

# Tenancy Configuration
TENANCY_BASE_DOMAIN=yourdomain.com
TENANCY_HEADER=X-Tenant
TENANCY_ALLOW_HEADER=true

# Session Domain (if using cookies)
SESSION_DOMAIN=.yourdomain.com

api/config/cors.php

Update allowed origins:

'allowed_origins' => [
    'https://app.yourdomain.com',
    'https://yourdomain.com',
    'https://*.yourdomain.com',
],

api/config/sanctum.php

Update stateful domains:

'stateful' => explode(',', env('SANCTUM_STATEFUL_DOMAINS', sprintf(
    '%s%s',
    'localhost,localhost:3000,localhost:5173,127.0.0.1,127.0.0.1:8000,::1',
    ',app.yourdomain.com,api.yourdomain.com'
))),

1.2 Web (React/Vite)

web/.env.production

VITE_API_BASE_URL=https://api.yourdomain.com/api/v1
VITE_TENANCY_HEADER=X-Tenant
VITE_TENANCY_VALUE=

1.3 Mobile Apps

mobile/packages/shared/src/api/client.js

Update the production API URL:

const API_BASE_URL = __DEV__ 
  ? 'http://10.0.2.2:8000/api/v1'      // Android emulator localhost
  : 'https://api.yourdomain.com/api/v1'; // Production

For iOS Simulator: Use http://localhost:8000/api/v1 instead of 10.0.2.2

2. Hostinger Shared Cloud Hosting Deployment

2.1 Prerequisites

  • Hostinger Cloud Hosting plan (Business or higher recommended)
  • SSH access enabled
  • PHP 8.2+ with required extensions
  • MySQL 8.0+ database
  • Composer installed
  • Node.js 18+ (for building web app locally)

2.2 API Deployment

Step 1: Create Subdomain

  1. Log in to Hostinger hPanel
  2. Go to Domains → Subdomains
  3. Create api.yourdomain.com
  4. Point it to public_html/api/public

Step 2: Upload Files (Using Git)

# SSH into your server
ssh u123456789@yourdomain.com

# Navigate to public_html
cd public_html

# Clone repository
git clone https://github.com/yourusername/gateflux.git api

# Or pull latest changes
cd api && git pull origin main

Step 3: Configure Environment

# SSH into server
cd public_html/api

# Copy environment file
cp .env.example .env

# Edit environment file
nano .env

Production .env Settings

APP_NAME=GateFlux
APP_ENV=production
APP_KEY=base64:YOUR_APP_KEY_HERE
APP_DEBUG=false
APP_URL=https://api.yourdomain.com

DB_CONNECTION=mysql
DB_HOST=localhost
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_database_user
DB_PASSWORD=your_database_password

TENANCY_BASE_DOMAIN=yourdomain.com
TENANCY_HEADER=X-Tenant
TENANCY_ALLOW_HEADER=true

CACHE_DRIVER=file
QUEUE_CONNECTION=database
SESSION_DRIVER=database

MAIL_MAILER=smtp
MAIL_HOST=smtp.hostinger.com
MAIL_PORT=465
MAIL_USERNAME=your_email@yourdomain.com
MAIL_PASSWORD=your_email_password
MAIL_ENCRYPTION=ssl
MAIL_FROM_ADDRESS=noreply@yourdomain.com
MAIL_FROM_NAME="GateFlux"

Step 4: Install Dependencies & Setup

# Install PHP dependencies
composer install --optimize-autoloader --no-dev

# Generate application key (if not set)
php artisan key:generate

# Run migrations
php artisan migrate --force

# Seed database (if needed)
php artisan db:seed --force

# Optimize for production
php artisan config:cache
php artisan route:cache
php artisan view:cache

# Set permissions
chmod -R 775 storage bootstrap/cache

Step 5: Configure .htaccess

public_html/api/.htaccess

<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteRule ^(.*)$ public/$1 [L]
</IfModule>

2.3 Web Deployment

Step 1: Build Locally

cd web

# Install dependencies
npm install

# Create production environment
cp .env.example .env.production
# Edit .env.production with production API URL

# Build for production
npm run build

Step 2: Upload Built Files

  1. Create subdomain app.yourdomain.com pointing to public_html/app
  2. Upload contents of web/dist/ to public_html/app/

Step 3: Configure .htaccess for SPA

public_html/app/.htaccess

<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteBase /
    
    # Handle Authorization Header
    RewriteCond %{HTTP:Authorization} ^(.*)
    RewriteRule .* - [e=HTTP_AUTHORIZATION:%1]
    
    # Redirect non-existing files to index.html
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule ^ index.html [L]
</IfModule>

# Enable GZIP compression
<IfModule mod_deflate.c>
    AddOutputFilterByType DEFLATE text/html text/css application/javascript application/json
</IfModule>

# Cache static assets
<IfModule mod_expires.c>
    ExpiresActive On
    ExpiresByType image/png "access plus 1 year"
    ExpiresByType image/jpg "access plus 1 year"
    ExpiresByType text/css "access plus 1 month"
    ExpiresByType application/javascript "access plus 1 month"
</IfModule>

2.4 Database Setup

  1. Go to Databases → MySQL Databases in hPanel
  2. Create new database: gateflux_prod
  3. Create database user with strong password
  4. Grant all privileges to the user
  5. Note down credentials for .env

Import Existing Data

# Export from local
mysqldump -u root -p gateflux > gateflux_backup.sql

# Import on Hostinger (via SSH or phpMyAdmin)
mysql -u your_db_user -p your_database < gateflux_backup.sql

3. Building Mobile APKs

3.1 Prerequisites

  • Node.js 18+
  • Java Development Kit (JDK) 17
  • Android Studio with SDK
  • Android SDK Platform 34 (or target version)
  • Android SDK Build-Tools 34.0.0

Environment Variables (Windows PowerShell)

$env:ANDROID_HOME = "C:\Users\YourUser\AppData\Local\Android\Sdk"
$env:JAVA_HOME = "C:\Program Files\Java\jdk-17"
$env:Path += ";$env:ANDROID_HOME\platform-tools;$env:ANDROID_HOME\tools"

Environment Variables (macOS/Linux)

export ANDROID_HOME=$HOME/Android/Sdk
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk
export PATH=$PATH:$ANDROID_HOME/platform-tools:$ANDROID_HOME/tools

3.2 Resident App APK

# Navigate to mobile directory
cd mobile

# Install dependencies
yarn install

# Navigate to resident app
cd apps/resident/android

# Clean previous builds
./gradlew clean

# Build debug APK
./gradlew assembleDebug

# Build release APK
./gradlew assembleRelease

APK Locations

  • Debug: apps/resident/android/app/build/outputs/apk/debug/app-debug.apk
  • Release: apps/resident/android/app/build/outputs/apk/release/app-release.apk

3.3 Security App APK

# Navigate to security app
cd mobile/apps/security/android

# Clean and build
./gradlew clean
./gradlew assembleDebug
./gradlew assembleRelease

3.4 Signed Release APK

For Play Store distribution, you need a signed APK.

Step 1: Generate Keystore

keytool -genkeypair -v -storetype PKCS12 -keystore gateflux-release.keystore -alias gateflux-key -keyalg RSA -keysize 2048 -validity 10000
⚠️ Important: Store the keystore file securely! You cannot recover it if lost.

Step 2: Configure Signing

apps/resident/android/gradle.properties

MYAPP_RELEASE_STORE_FILE=gateflux-release.keystore
MYAPP_RELEASE_KEY_ALIAS=gateflux-key
MYAPP_RELEASE_STORE_PASSWORD=your_store_password
MYAPP_RELEASE_KEY_PASSWORD=your_key_password

apps/resident/android/app/build.gradle

Update the signingConfigs block:

signingConfigs {
    debug {
        storeFile file('debug.keystore')
        storePassword 'android'
        keyAlias 'androiddebugkey'
        keyPassword 'android'
    }
    release {
        if (project.hasProperty('MYAPP_RELEASE_STORE_FILE')) {
            storeFile file(MYAPP_RELEASE_STORE_FILE)
            storePassword MYAPP_RELEASE_STORE_PASSWORD
            keyAlias MYAPP_RELEASE_KEY_ALIAS
            keyPassword MYAPP_RELEASE_KEY_PASSWORD
        }
    }
}
buildTypes {
    release {
        signingConfig signingConfigs.release
        minifyEnabled true
        shrinkResources true
        proguardFiles getDefaultProguardFile("proguard-android-optimize.txt"), "proguard-rules.pro"
    }
}

Step 3: Build Signed APK

cd apps/resident/android
./gradlew assembleRelease

Step 4: Build AAB for Play Store

./gradlew bundleRelease

AAB Location: app/build/outputs/bundle/release/app-release.aab

Update App Version

Before each release, update version in build.gradle:

defaultConfig {
    applicationId "com.gatefluxresident"
    minSdkVersion rootProject.ext.minSdkVersion
    targetSdkVersion rootProject.ext.targetSdkVersion
    versionCode 2          // Increment for each release
    versionName "1.1.0"    // Semantic version
}

4. SSL Configuration

Hostinger provides free SSL via Let's Encrypt.

  1. Go to SSL in hPanel
  2. Select your domains:
    • yourdomain.com
    • api.yourdomain.com
    • app.yourdomain.com
  3. Click Install SSL
  4. Enable Force HTTPS

Force HTTPS in .htaccess

RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

6. Troubleshooting

API Issues

500 Internal Server Error

# Check Laravel logs
tail -f storage/logs/laravel.log

# Fix permissions
chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data storage bootstrap/cache

CORS Errors

  • Check config/cors.php allowed origins
  • Verify api/.htaccess has correct headers
  • Clear config cache: php artisan config:clear

Database Connection Failed

  • Verify credentials in .env
  • Check if database exists
  • Ensure MySQL user has proper privileges

Web Issues

Blank Page

  • Check browser console for errors
  • Verify API URL in .env.production
  • Check .htaccess SPA routing

Mobile Build Issues

Gradle Build Failed

# Clean and rebuild
cd android
./gradlew clean
./gradlew assembleDebug --stacktrace

SDK Not Found

  • Verify ANDROID_HOME environment variable
  • Install required SDK platforms via Android Studio

Metro Bundler Issues

# Reset Metro cache
npx react-native start --reset-cache

Checklist Before Going Live

  • Update all domain references in code
  • Set APP_DEBUG=false in production
  • Configure SSL certificates
  • Set up database backups
  • Configure email (SMTP)
  • Test all API endpoints
  • Test web app in production
  • Build and test both mobile APKs
  • Set up error monitoring (optional)
  • Configure queue workers (if using)
  • Set up cron jobs for scheduled tasks
