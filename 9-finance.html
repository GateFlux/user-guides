Finance & Expenses Guide

Manage expenses, budgets, and approval workflows with Treasurer and President oversight.

Overview

GateFlux provides comprehensive financial management tools:

  • Expense Management - Track and categorize all society expenses
  • Approval Workflow - Multi-level approval with configurable thresholds
  • Budget Tracking - Budget vs actual comparison
  • Treasurer Dashboard - Consolidated financial overview

Expense Approval Workflow

Dual Approval System

High-value expenses require approval from both Treasurer and President:

Basic Approval

Expenses below threshold (default ₹50,000) require only Treasurer approval.

Dual Approval

High-value expenses require both Treasurer AND President approval before payment.

Approval Thresholds

Amount Range Required Approvals Approvers
₹0 - ₹10,000 1 Treasurer
₹10,001 - ₹50,000 1 Treasurer
₹50,001+ 2 Treasurer + President

Managing Expenses

Creating an Expense

HTTP 
POST /api/v1/expenses
Authorization: Bearer <token>
Content-Type: application/json

{
    "category_id": "uuid-of-category",
    "description": "Monthly security guard salary",
    "amount": 45000,
    "expense_date": "2026-02-15",
    "vendor_id": "uuid-of-vendor",
    "payment_method": "bank_transfer",
    "receipt_number": "RCP-2026-001",
    "notes": "For 3 guards × ₹15,000 each"
}

Expense Categories

Retrieve available expense categories:

HTTP 
GET /api/v1/expenses/categories
Authorization: Bearer <token>

Common Categories:

  • Security & Maintenance
  • Utilities (Electricity, Water, Gas)
  • Repairs & Maintenance
  • Landscaping & Gardening
  • Administrative Expenses
  • Insurance & Legal

Submitting for Approval

After creating an expense, submit it for approval:

HTTP 
POST /api/v1/expenses/{expense_id}/submit
Authorization: Bearer <token>

Expense Approval

View Pending Approvals

Treasurers and Presidents can view expenses awaiting their approval:

HTTP 
GET /api/v1/expenses/pending-approval
Authorization: Bearer <token>

Approving an Expense

HTTP 
POST /api/v1/expenses/{expense_id}/approve
Authorization: Bearer <token>
Content-Type: application/json

{
    "notes": "Approved. Please process payment by month end."
}
Dual Approval Logic: For high-value expenses, the system automatically checks if both Treasurer and President have approved before changing status to "approved".

Rejecting an Expense

HTTP 
POST /api/v1/expenses/{expense_id}/reject
Authorization: Bearer <token>
Content-Type: application/json

{
    "reason": "Vendor quote is higher than market rate. Please get alternative quotes."
}

Approval Workflow States

Status Description
draft Expense created but not submitted
pending_approval Submitted and awaiting approval
approved All required approvals received
rejected Expense rejected by approver
paid Payment has been made

Recording Payments

After approval, record the payment:

HTTP 
POST /api/v1/expenses/{expense_id}/payment
Authorization: Bearer <token>
Content-Type: application/json

{
    "payment_date": "2026-02-20",
    "payment_reference": "UTR123456789",
    "payment_method": "bank_transfer"
}

Treasurer Dashboard

The Treasurer Dashboard provides a consolidated view of financial health:

HTTP 
GET /api/v1/treasurer/dashboard
Authorization: Bearer <token>

Dashboard includes:

  • Monthly expense summary
  • Pending approvals count
  • Budget utilization overview
  • Recent expense activity
  • Top expense categories
