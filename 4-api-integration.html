API Integration Guide

Complete reference for integrating with the GateFlux REST API.

Overview

This guide covers integrating with the GateFlux REST API. The API follows RESTful conventions and uses JSON for request and response payloads.

Base URL

Environment Base URL
Developmenthttp://localhost:8000/api
Staginghttps://staging-api.gateflux.co/api
Productionhttps://api.gateflux.co/api

Authentication

GateFlux uses Laravel Sanctum for token-based authentication.

Login

HTTP 
POST /api/auth/login
Content-Type: application/json

{
    "email": "user@example.com",
    "password": "your_password",
    "device_name": "web_browser"
}

Response:

JSON 
{
    "success": true,
    "data": {
        "user": {
            "id": 1,
            "name": "John Doe",
            "email": "user@example.com",
            "role": "society_admin"
        },
        "token": "1|abc123xyz456...",
        "token_type": "Bearer",
        "expires_at": "2024-03-16T10:30:00Z"
    }
}

Using the Token

Include the token in all subsequent requests:

HTTP 
GET /api/societies
Authorization: Bearer 1|abc123xyz456...
Content-Type: application/json

Logout

HTTP 
POST /api/auth/logout
Authorization: Bearer <token>

Webhooks

GateFlux can send webhooks for important events.

Configure Webhooks

HTTP 
POST /api/societies/{society_id}/webhooks
Authorization: Bearer <admin_token>
Content-Type: application/json

{
    "url": "https://your-server.com/webhook",
    "events": ["visitor.checked_in", "visitor.checked_out", "complaint.created"],
    "secret": "your_webhook_secret"
}

Webhook Events

Event Description
visitor.pre_approvedVisitor pre-approval created
visitor.checked_inVisitor checked in
visitor.checked_outVisitor checked out
visitor.rejectedVisitor entry rejected
complaint.createdNew complaint filed
complaint.resolvedComplaint resolved
announcement.createdNew announcement posted

Webhook Payload

JSON 
{
    "event": "visitor.checked_in",
    "timestamp": "2024-02-16T10:30:00Z",
    "data": {
        "visitor_id": 12345,
        "flat_id": 101,
        "visitor_name": "John Visitor",
        "entry_time": "2024-02-16T10:30:00Z"
    },
    "signature": "sha256=abc123..."
}

Verifying Webhook Signature

PHP 
$payload = file_get_contents('php://input');
$signature = $_SERVER['HTTP_X_GATEFLUX_SIGNATURE'];
$secret = 'your_webhook_secret';

$expected = 'sha256=' . hash_hmac('sha256', $payload, $secret);

if (hash_equals($expected, $signature)) {
    // Valid webhook
}

Read operations60 requests/minute

X-RateLimit-Reset: 1708089600

Error Handling

Common Error Codes

Code Message Solution
AUTH_INVALID_TOKENInvalid or expired tokenRe-authenticate
AUTH_INSUFFICIENT_PERMISSIONSInsufficient permissionsCheck user role
RESOURCE_NOT_FOUNDResource not foundVerify resource ID
VALIDATION_ERRORValidation failedCheck request payload
RATE_LIMIT_EXCEEDEDToo many requestsWait and retry

Retry Strategy

For 429 (Rate Limit) and 5xx errors, implement exponential backoff:

JavaScript 
async function apiRequest(url, options, retries = 3) {
    for (let i = 0; i < retries; i++) {
        try {
            const response = await fetch(url, options);
            if (response.ok) return response.json();
            if (response.status === 429 || response.status >= 500) {
                await sleep(Math.pow(2, i) * 1000);
                continue;
            }
            throw new Error(`HTTP ${response.status}`);
        } catch (error) {
            if (i === retries - 1) throw error;
        }
    }
}
