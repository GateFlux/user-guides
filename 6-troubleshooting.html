Troubleshooting Guide

Diagnose and resolve common issues with GateFlux installations and operations.

Installation Issues

API (Laravel) Issues

Composer Install Fails

Error:

Your requirements could not be resolved to an installable set of packages.

Solution:

Bash 
# Update composer
composer self-update

# Clear cache and retry
composer clear-cache
composer install --no-cache

Database Connection Error

Error:

SQLSTATE[HY000] [2002] Connection refused

Solutions:

  1. Verify MySQL is running: 
    # Windows
net start mysql

# Linux/Mac
sudo systemctl status mysql
  2. Check .env database settings: 
    DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=gateflux
DB_USERNAME=root
DB_PASSWORD=your_password
  3. Create database if it doesn't exist: 
    CREATE DATABASE gateflux CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

Migration Errors

Error:

SQLSTATE[42S01]: Base table or view already exists

Solution:

Bash 
# Fresh migration (WARNING: Deletes all data)
php artisan migrate:fresh

# Or rollback and re-run
php artisan migrate:rollback
php artisan migrate

Permission Errors

Error:

The stream or file "storage/logs/laravel.log" could not be opened

Solution:

Bash 
# Linux/Mac
sudo chmod -R 775 storage bootstrap/cache
sudo chown -R www-data:www-data storage bootstrap/cache

# Windows (run as Administrator)
icacls storage /grant Users:F /T
icacls bootstrap\cache /grant Users:F /T

Web Application Issues

npm Install Fails

Error:

npm ERR! ERESOLVE unable to resolve dependency tree

Solution:

Bash 
# Clear npm cache
npm cache clean --force

# Delete node_modules and lock file
rm -rf node_modules package-lock.json

# Install with legacy peer deps
npm install --legacy-peer-deps

Build Errors

Error:

Error: Cannot find module 'vite'

Solution:

Bash 
# Reinstall dependencies
rm -rf node_modules
npm install

# If still failing, install vite globally
npm install -g vite

CORS Errors

Error in browser console:

Access to XMLHttpRequest blocked by CORS policy

Solution:

  1. Check API .env: 
    SANCTUM_STATEFUL_DOMAINS=localhost:5173,localhost:3000
SESSION_DOMAIN=localhost
  2. Update api/config/cors.php: 
    'allowed_origins' => ['http://localhost:5173', 'http://localhost:3000'],
'supports_credentials' => true,

Mobile Application Issues

iOS Pod Install Fails

Error:

[!] CocoaPods could not find compatible versions

Solution:

Bash 
cd ios
rm -rf Pods Podfile.lock
pod repo update
pod install

Android Build Fails

Error:

Could not find com.android.tools.build:gradle

Solution:

  1. Update android/build.gradle: 
    classpath 'com.android.tools.build:gradle:8.1.0'
  2. Clean and rebuild: 
    cd android
./gradlew clean
cd ..
yarn android

Metro Bundler Issues

Error:

Unable to resolve module

Solution:

Bash 
# Clear Metro cache
npx react-native start --reset-cache

# Or
yarn start --reset-cache

Runtime Issues

Authentication Problems

Token Invalid or Expired

Symptoms:

  • 401 Unauthorized responses
  • Automatic logout

Solutions:

  1. Check token expiration in API: 
    // config/sanctum.php
'expiration' => 60 * 24, // 24 hours
  2. Implement token refresh in client: 
    axios.interceptors.response.use(
    response => response,
    async error => {
        if (error.response.status === 401) {
            await refreshToken();
            return axios(error.config);
        }
        throw error;
    }
);

Login Fails

Error:

JSON 
{
    "message": "These credentials do not match our records."
}

Solutions:

  1. Verify user exists: 
    php artisan tinker
>>> User::where('email', 'user@example.com')->first()
  2. Reset password: 
    php artisan tinker
>>> $user = User::where('email', 'user@example.com')->first();
>>> $user->password = Hash::make('newpassword');
>>> $user->save();

Two-Factor Authentication Issues

Symptoms:

  • 2FA codes not working
  • User locked out after losing 2FA device
  • QR code not scanning

Solutions:

  1. 2FA code not working - Check server time sync:

    TOTP codes are time-sensitive. Ensure server time is synchronized:

    # Check server time
date
# Sync with NTP
timedatectl set-ntp true
  2. User locked out - Reset 2FA via database: 
    php artisan tinker
>>> $user = User::where('email', 'user@example.com')->first();
>>> $user->two_factor_secret = null;
>>> $user->two_factor_enabled = false;
>>> $user->two_factor_recovery_codes = null;
>>> $user->two_factor_confirmed_at = null;
>>> $user->save();
  3. QR code not scanning - Manual entry:

    Users can manually enter the secret key shown below the QR code into their authenticator app instead of scanning.

  4. Recovery codes not working:
    • Check the code hasn't been used before (each code is single-use)
    • Verify no extra spaces or characters were copied
    • Recovery codes are case-sensitive

Notification Issues

Push Notifications Not Received

Checklist:

  1. Verify Firebase configuration: 
    # Test Firebase connection
php artisan firebase:test
  2. Check device token is saved: 
    SELECT * FROM device_tokens WHERE user_id = 1;
  3. Verify notification permissions on device

SMS Not Sending

Checklist:

  1. Verify SMS provider credentials: 
    php artisan tinker
>>> config('services.twilio')
  2. Check SMS logs: 
    SELECT * FROM sms_logs ORDER BY created_at DESC LIMIT 10;
  3. Test SMS sending: 
    php artisan sms:test +919876543210 "Test message"

Performance Issues

Slow API Response

Diagnosis:

Bash 
php artisan tinker
>>> DB::enableQueryLog();
>>> // Make API request
>>> DB::getQueryLog();

Solutions:

  1. Add database indexes: 
    Schema::table('visitors', function ($table) {
    $table->index('society_id');
    $table->index('created_at');
});
  2. Use eager loading: 
    // Instead of
$visitors = Visitor::all();
foreach ($visitors as $visitor) {
    echo $visitor->flat->number;
}

// Use
$visitors = Visitor::with('flat')->get();
  3. Enable caching: 
    php artisan config:cache
php artisan route:cache

High Memory Usage

Solutions:

  1. Process large datasets in chunks: 
    Visitor::chunk(100, function ($visitors) {
    foreach ($visitors as $visitor) {
        // Process
    }
});
  2. Clear old sessions: 
    php artisan session:clear
  3. Optimize autoloader: 
    composer install --optimize-autoloader --no-dev

Logging & Debugging

Enable Debug Mode

Warning

DO NOT use debug mode in production!

ENV 
APP_DEBUG=true
LOG_LEVEL=debug

View Logs

Bash 
# View latest logs
tail -f storage/logs/laravel.log

# Search for errors
grep -i "error" storage/logs/laravel.log

Debug API Requests

Use Laravel Telescope (development only):

Bash 
composer require laravel/telescope --dev
php artisan telescope:install
php artisan migrate

Access at: http://localhost:8000/telescope
